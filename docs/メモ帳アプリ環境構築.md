# メモ帳アプリ 環境構築手順

React（フロントエンド）+ Express（バックエンド）+ Supabase（データベース）を使ったメモ帳アプリの構築手順です。

---

> ⚠️ **この端末での注意事項（社内ネットワーク環境）**
>
> 社内プロキシが HTTPS 通信を SSL インスペクション（中間者検査）するため、Node.js から Supabase への接続が `TypeError: fetch failed` で失敗します。
> Node.js は Windows の証明書ストアではなく独自のルート証明書を使うため、社内プロキシの証明書を信頼できないことが原因です。
>
> **対処：** `server/index.js` の先頭に以下を追加してください（開発環境のみ無効化）。
>
> ```js
> if (process.env.NODE_ENV !== 'production') {
>   process.env.NODE_TLS_REJECT_UNAUTHORIZED = '0';
> }
> ```
>
> **注意：** これは開発環境限定の対処です。本番環境（`NODE_ENV=production`）では自動的にスキップされます。
> 正式な対処は IT 部門から社内 CA 証明書を取得し、`NODE_EXTRA_CA_CERTS` 環境変数に設定することです。

## 2️⃣ バックエンド（Express）のセットアップ

```bash
mkdir server
cd server
npm init -y
npm install express cors dotenv @supabase/supabase-js multer express-rate-limit
npm install --save-dev nodemon
```

### `server/index.js` を編集

以下の内容を実装してください。

- 開発環境のみ TLS 検証を無効化（`process.env.NODE_ENV !== 'production'` の条件付き）
- `dotenv` の読み込み
- Express アプリの初期化（`cors`、`express.json()` ミドルウェア適用）
  - `cors` の `origin` は環境変数 `ALLOWED_ORIGIN` から取得（デフォルト: `http://localhost:5173`）
- レートリミットの設定（`express-rate-limit`）
  - 通常 API: 15分あたり100リクエスト
  - 画像アップロード: 15分あたり20リクエスト
- Supabase クライアントの初期化
  - 通常クライアント（`SUPABASE_URL` + `SUPABASE_KEY`）
  - 管理クライアント（`SUPABASE_SERVICE_KEY`）— Storage 削除の RLS バイパス用
- `multer` の設定（メモリストレージ、5MB 上限、画像ファイルのみ許可）
- 以下の API エンドポイントの実装：
  - `POST /api/upload` — 画像を Supabase Storage にアップロードし公開URLを返す
  - `GET /api/memos` — メモ一覧の取得（`created_at` 降順）
  - `POST /api/memos` — メモの作成（`title`・`content`・`image_url`・`author`）
  - `PUT /api/memos/:id` — メモの更新（画像差し替え時は旧ファイルをStorageから削除）
  - `DELETE /api/memos/:id` — メモの削除（Storageの画像も合わせて削除）
- 入力バリデーション：タイトル50文字以内・本文500文字以内・名前20文字以内

### `server/package.json` のスクリプトを編集

`scripts` に `start`（`node index.js`）と `dev`（`nodemon index.js`）を追加してください。

### `server/.env` を作成（後でSupabaseの値を入力）

```env
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_supabase_anon_key
SUPABASE_SERVICE_KEY=your_supabase_service_role_key
PORT=3001
ALLOWED_ORIGIN=http://localhost:5173
```

`SUPABASE_SERVICE_KEY` は Supabase コンソールの「Project Settings」→「API」→「service_role」欄にある Secret キーです。Storage からの画像削除（RLS のバイパス）に使用します。

`ALLOWED_ORIGIN` は CORS で許可するフロントエンドのオリジンです。本番環境では Render 等でデプロイされたフロントエンドの URL に変更してください。

> ⚠️ service_role キーは強力な権限を持ちます。`.env` はGitにコミットしないでください。

### `server/.gitignore` を作成

```
node_modules/
.env
```

## 3️⃣ フロントエンド（React）のセットアップ

`memo-app/` ルートに戻って：

```bash
cd ..
npm create vite@latest client -- --template react
cd client
npm install
npm install axios lucide-react
```

### `client/src/` 以下のファイル構成

```
client/src/
├── main.tsx                  # エントリーポイント（変更不要）
├── App.tsx                   # メインコンポーネント（テーマ状態管理・レイアウト）
├── App.css                   # App コンポーネントのスタイル
├── types.ts                  # Memo 型定義・日付フォーマット関数
├── styles.ts                 # テーマカラー定義・スタイルユーティリティ
├── index.css                 # グローバルスタイル
├── components/
│   ├── Header.tsx            # ヘッダー（タイトル・テーマ切り替えボタン）
│   ├── MemoCard.tsx          # メモカード（一覧表示の1件分）
│   ├── MemoForm.tsx          # メモ投稿・編集フォーム
│   ├── Modal.tsx             # メモ詳細モーダル
│   └── ViewToggle.tsx        # グリッド/リスト表示切り替えボタン
└── hooks/
    ├── useMemos.ts           # メモ CRUD・画像アップロードのカスタムフック
    └── useForm.ts            # フォームの入力状態管理カスタムフック
```

以下の内容を実装してください。

- `types.ts`: `Memo` インターフェース（`id`・`title`・`content`・`image_url`・`author`・`created_at`・`updated_at`）と `formatDate` 関数
- `styles.ts`: テーマカラーの定義（`themes`・`themeList`・`getInputStyle`）。テーマは `green`・`blue`・`red`・`yellow`・`mono` の5種類
- `hooks/useMemos.ts`: Axios を使ったメモ一覧取得・追加・更新・削除・画像アップロード。ベース URL は環境変数 `VITE_API_URL`（未設定時は `http://localhost:3001/api`）から取得
- `hooks/useForm.ts`: フォームの入力値（title・content・author・image_url）の状態管理
- `components/Header.tsx`: アプリタイトルとテーマ切り替えボタン
- `components/MemoCard.tsx`: 一覧に表示するカード。クリックで詳細モーダルを開く
- `components/MemoForm.tsx`: メモ投稿・編集フォーム。画像選択・プレビューも含む
- `components/Modal.tsx`: メモ詳細を表示するモーダル（編集・削除操作を含む）
- `components/ViewToggle.tsx`: グリッド/リスト表示の切り替えボタン
- `App.tsx`: 上記コンポーネントを組み合わせたメインコンポーネント

## 4️⃣ Supabase のセットアップ

1. [https://supabase.com](https://supabase.com) にアクセスし、無料アカウントを作成
2. 新しいプロジェクトを作成
3. 左メニュー「Table Editor」→「New table」でテーブルを作成

### テーブル定義の例（`memos`）

> 以下はテーブル構成の一例です。アプリの要件に合わせて適宜変更してください。

| カラム名     | 型          | デフォルト | 備考             |
| ------------ | ----------- | ---------- | ---------------- |
| `id`         | int8        | 自動生成   | Primary Key      |
| `title`      | text        | -          | メモのタイトル（50文字以内） |
| `content`    | text        | `''`       | メモの本文（500文字以内） |
| `image_url`  | text        | `null`     | 添付画像のURL    |
| `author`     | text        | `null`     | 投稿者名（20文字以内） |
| `created_at` | timestamptz | `now()`    | 作成日時         |
| `updated_at` | timestamptz | `null`     | 更新日時         |

または「SQL Editor」で以下のような SQL を実行して作成することもできます（カラム構成はアプリに合わせて調整してください）：

```sql
create table memos (
  id bigint generated always as identity primary key,
  title text not null,
  content text default '',
  image_url text,
  author text,
  created_at timestamptz default now(),
  updated_at timestamptz
);
```

既存の `memos` テーブルにカラムを後から追加する場合は SQL Editor で以下を実行：

```sql
alter table memos add column image_url text;
alter table memos add column author text;
```

4. 「Project Settings」→「API」から以下をコピーし、`server/.env` に貼り付け：
   - **Project URL** → `SUPABASE_URL`
   - **anon public key** → `SUPABASE_KEY`
   - **service_role key** → `SUPABASE_SERVICE_KEY`

### Supabase Storage のセットアップ（画像添付機能）

1. Supabase コンソール左メニュー「Storage」→「New bucket」
2. 以下を入力して作成：
   - **Name**: `memo-images`
   - **Public bucket**: ✅ オン（公開URLで画像を表示するため）
3. 作成後、「Policies」タブで削除ポリシーを追加（`SUPABASE_SERVICE_KEY` を使う場合は不要）

> `SUPABASE_SERVICE_KEY` を `.env` に設定している場合、RLS をバイパスするため Storage のポリシー設定は不要です。設定しない場合は以下の SQL を「SQL Editor」で実行してください。

```sql
-- Storage への書き込みを許可
create policy "Allow insert" on storage.objects
  for insert with check (bucket_id = 'memo-images');

-- Storage からの削除を許可
create policy "Allow delete" on storage.objects
  for delete using (bucket_id = 'memo-images');
```

## 7️⃣ Render へのデプロイ

[https://render.com](https://render.com) に登録し、GitHubと連携します。

### バックエンドのデプロイ

1. 「New +」→「Web Service」
2. リポジトリを選択
3. 以下を入力：
   - **Root Directory**: `server`
   - **Build Command**: `npm install`
   - **Start Command**: `node index.js`
4. 「Environment Variables」に `.env` の内容を入力（`ALLOWED_ORIGIN` はフロントエンドの Render URL に変更）
5. 「Create Web Service」でデプロイ完了

### フロントエンドのデプロイ

1. 「New +」→「Static Site」
2. リポジトリを選択
3. 以下を入力：
   - **Root Directory**: `client`
   - **Build Command**: `npm run build`
   - **Publish Directory**: `dist`
4. 「Environment Variables」に以下を追加：
   - `VITE_API_URL` = Renderで発行されたバックエンドのURL

5. `client/src/App.tsx` の API ベース URL をRenderのバックエンドURLに変更してください。環境変数 `VITE_API_URL` から取得するよう実装するのが推奨です。

## 🛠️ トラブルシューティング

| 症状                              | 原因・対処                                                                      |
| --------------------------------- | ------------------------------------------------------------------------------- |
| `CORS error`                      | `server/.env` の `ALLOWED_ORIGIN` がフロントエンドのオリジンと一致しているか確認 |
| `Cannot GET /api/memos`           | サーバーが起動しているか確認（ターミナル①）                                      |
| Supabase接続エラー                | `.env` の URL と KEY が正しいか確認                                             |
| 画像アップロードで500エラー       | Storage バケット `memo-images` が作成済みか確認。バケット名のタイポにも注意     |
| 画像削除してもStorageに残る       | `SUPABASE_SERVICE_KEY` が `.env` に設定されているか確認。未設定の場合はStorageのポリシーを追加する |
| Renderでスリープ                  | 無料プランは15分無操作でスリープ。有料プランで解消                              |
