# メモ帳アプリ 環境構築手順

React（フロントエンド）+ Express（バックエンド）+ Supabase（データベース）を使ったメモ帳アプリの構築手順です。

---

> ⚠️ **この端末での注意事項（社内ネットワーク環境）**
>
> 社内プロキシが HTTPS 通信を SSL インスペクション（中間者検査）するため、Node.js から Supabase への接続が `TypeError: fetch failed` で失敗します。
> Node.js は Windows の証明書ストアではなく独自のルート証明書を使うため、社内プロキシの証明書を信頼できないことが原因です。
>
> **対処：** `server/index.js` の先頭に以下を追加してください。
>
> ```js
> process.env.NODE_TLS_REJECT_UNAUTHORIZED = "0";
> ```
>
> または `server/.env` に以下を追加し、サーバー起動前に読み込まれるようにしてください。
>
> ```
> NODE_TLS_REJECT_UNAUTHORIZED=0
> ```
>
> **注意：** これは開発環境限定の対処です。本番環境では使用しないでください。
> 正式な対処は IT 部門から社内 CA 証明書を取得し、`NODE_EXTRA_CA_CERTS` 環境変数に設定することです。

## 📁 ディレクトリ構成

```
memo-app/
├── client/          # React フロントエンド
└── server/          # Express バックエンド
```

## 🔧 前提条件

以下がインストールされていることを確認してください。

- **Node.js** v18以上 → https://nodejs.org/
- **npm** v9以上（Node.jsに同梱）
- **Git** → https://git-scm.com/

バージョン確認：

```bash
node -v
npm -v
git --version
```

## 1️⃣ プロジェクトの作成

```bash
mkdir memo-app
cd memo-app
```

---

## 2️⃣ バックエンド（Express）のセットアップ

```bash
mkdir server
cd server
npm init -y
npm install express cors dotenv @supabase/supabase-js
npm install --save-dev nodemon
```

### `server/index.js` を編集

以下の内容を実装してください。

- `dotenv` の読み込み
- Express アプリの初期化（`cors`、`express.json()` ミドルウェア適用）
- Supabase クライアントの初期化（`SUPABASE_URL`、`SUPABASE_KEY` は環境変数から取得）
- 以下の API エンドポイントの実装：
  - `GET /api/memos` — メモ一覧の取得
  - `POST /api/memos` — メモの作成
  - `PUT /api/memos/:id` — メモの更新
  - `DELETE /api/memos/:id` — メモの削除

### `server/package.json` のスクリプトを編集

`scripts` に `start`（`node index.js`）と `dev`（`nodemon index.js`）を追加してください。

### `server/.env` を作成（後でSupabaseの値を入力）

```env
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_supabase_anon_key
PORT=3001
```

> ⚠️ `.env` はGitにコミットしないでください。

### `server/.gitignore` を作成

```
node_modules/
.env
```

---

## 3️⃣ フロントエンド（React）のセットアップ

`memo-app/` ルートに戻って：

```bash
cd ..
npm create vite@latest client -- --template react
cd client
npm install
npm install axios
```

### `client/src/App.tsx` を編集

以下の内容を実装してください。

- バックエンド API への接続（ベース URL: `http://localhost:3001/api`）
- メモ一覧の取得・表示
- メモの新規作成フォーム（タイトル・本文の入力）
- メモの編集・削除機能

---

## 4️⃣ Supabase のセットアップ

1. [https://supabase.com](https://supabase.com) にアクセスし、無料アカウントを作成
2. 新しいプロジェクトを作成
3. 左メニュー「Table Editor」→「New table」でテーブルを作成

### テーブル定義の例（`memos`）

> 以下はテーブル構成の一例です。アプリの要件に合わせて適宜変更してください。

| カラム名     | 型          | デフォルト | 備考           |
| ------------ | ----------- | ---------- | -------------- |
| `id`         | int8        | 自動生成   | Primary Key    |
| `title`      | text        | -          | メモのタイトル |
| `content`    | text        | `''`       | メモの本文     |
| `created_at` | timestamptz | `now()`    | 作成日時       |

または「SQL Editor」で以下のような SQL を実行して作成することもできます（カラム構成はアプリに合わせて調整してください）：

```sql
create table memos (
  id bigint generated always as identity primary key,
  title text not null,
  content text default '',
  created_at timestamptz default now()
);
```

4. 「Project Settings」→「API」から以下をコピーし、`server/.env` に貼り付け：
   - **Project URL** → `SUPABASE_URL`
   - **anon public key** → `SUPABASE_KEY`

## 7️⃣ Render へのデプロイ

[https://render.com](https://render.com) に登録し、GitHubと連携します。

### バックエンドのデプロイ

1. 「New +」→「Web Service」
2. リポジトリを選択
3. 以下を入力：
   - **Root Directory**: `server`
   - **Build Command**: `npm install`
   - **Start Command**: `node index.js`
4. 「Environment Variables」に `.env` の内容を入力
5. 「Create Web Service」でデプロイ完了

### フロントエンドのデプロイ

1. 「New +」→「Static Site」
2. リポジトリを選択
3. 以下を入力：
   - **Root Directory**: `client`
   - **Build Command**: `npm run build`
   - **Publish Directory**: `dist`
4. 「Environment Variables」に以下を追加：
   - `VITE_API_URL` = Renderで発行されたバックエンドのURL

5. `client/src/App.tsx` の API ベース URL をRenderのバックエンドURLに変更してください。環境変数 `VITE_API_URL` から取得するよう実装するのが推奨です。

## 🛠️ トラブルシューティング

| 症状                    | 原因・対処                                                 |
| ----------------------- | ---------------------------------------------------------- |
| `CORS error`            | `server/index.js` の `cors()` が正しく設定されているか確認 |
| `Cannot GET /api/memos` | サーバーが起動しているか確認（ターミナル①）                |
| Supabase接続エラー      | `.env` の URL と KEY が正しいか確認                        |
| Renderでスリープ        | 無料プランは15分無操作でスリープ。有料プランで解消         |
