## 1. アプリ全体の構成を理解する

このアプリは「3層構造」で作られています。

```
ブラウザ（フロントエンド）
     ↕  HTTP通信（API）
サーバー（バックエンド）
     ↕  Supabase SDK
データベース（Supabase）
```

| 層             | 担当                      | 使用技術                  | 場所      |
| -------------- | ------------------------- | ------------------------- | --------- |
| フロントエンド | 画面表示・ユーザー操作    | React / TypeScript / Vite | `client/` |
| バックエンド   | ビジネスロジック・API提供 | Node.js / Express         | `server/` |
| データベース   | データの永続保存          | Supabase（PostgreSQL）    | クラウド  |

**なぜ3層に分けるのか？**

- 画面（フロントエンド）と処理（バックエンド）を分けることで、それぞれを独立して変更・改善できます
- データベースに直接アクセスするのはサーバーだけにすることで、セキュリティが高まります

## 2. フロントエンド：React + TypeScript + Vite

### React とは

**React** は Facebook（現 Meta）が作った JavaScript ライブラリです。画面の部品（コンポーネント）を組み合わせて UI を作ります。

#### コンポーネントとは

画面の「部品」のことです。このアプリでは機能ごとに複数のコンポーネントに分割されています。

```
client/src/
├── App.tsx                  ← ルートコンポーネント（全体の組み立て役）
├── components/
│   ├── Header.tsx           ← ヘッダー（タイトル・テーマ切替・ユーザー名入力）
│   ├── MemoForm.tsx         ← メモ入力フォーム（新規追加・編集共用）
│   ├── ViewToggle.tsx       ← グリッド/リスト切替ボタン
│   ├── MemoCard.tsx         ← メモ1件の表示カード
│   └── Modal.tsx            ← メモ詳細のモーダルダイアログ
├── hooks/
│   ├── useMemos.ts          ← メモ一覧の取得・追加・更新・削除・画像アップロード
│   └── useForm.ts           ← 投稿フォームの入力状態と操作
├── types.ts                 ← 型定義・ユーティリティ関数
└── styles.ts                ← テーマ色の定義
```

`App.tsx` はコンポーネントを組み合わせるだけのシンプルな構造になっています。

```tsx
// client/src/App.tsx
function App() {
  // ...
  return (
    <div>
      <Header ... />
      <MemoForm ... />
      <ViewToggle ... />
      {memos.map(memo => <MemoCard key={memo.id} ... />)}
      {detailMemo && <Modal ... />}
    </div>
  );
}
```

> **JSX とは**: JavaScript の中に HTML っぽい記法で書ける構文です。`<h1>` などのタグや `<Header />` のような自作コンポーネントを直接 JS 内に書けます。

### useState とは（状態管理）

React では、変化するデータを「**state（ステート）**」として管理します。

```tsx
// client/src/App.tsx — テーマ・ユーザー名・表示モード・詳細表示の状態を管理
const [theme, setTheme] = useState<ThemeName>(
  () => (localStorage.getItem('memo-theme') as ThemeName) || 'green'
);
const [username, setUsername] = useState(
  () => localStorage.getItem('memo-username') || ''
);
const [viewMode, setViewMode] = useState<'grid' | 'list'>(
  () => (localStorage.getItem('memo-view') as 'grid' | 'list') || 'grid'
);
const [detailMemo, setDetailMemo] = useState<Memo | null>(null);
```

```tsx
// client/src/hooks/useForm.ts — フォームの入力状態と編集モードを管理
const [title, setTitle] = useState('');
const [content, setContent] = useState('');
const [editingId, setEditingId] = useState<number | null>(null);
```

```tsx
// client/src/hooks/useMemos.ts — メモ一覧の状態は専用のカスタムフックで管理
const [memos, setMemos] = useState<Memo[]>([]);
```

**読み方**: `[現在の値, 値を変える関数] = useState(初期値)`

- `memos`：現在保存されているメモの配列
- `setMemos`：`memos` を更新するための関数
- state が変わると、React は自動的に画面を再描画します

> **ポイント**: state は「どこで管理するか」が重要です。メモ一覧は `useMemos`、フォーム入力は `useForm`、テーマ・ユーザー名などアプリ全体の設定は `App` と、役割ごとに分散させています。

---

### useEffect とは（副作用）

**useEffect** は「コンポーネントが表示されたとき（マウント時）に実行したい処理」を書く場所です。

```tsx
// client/src/hooks/useMemos.ts
// fetchMemos は追加・更新・削除後の再取得でも使うため関数として定義している
const fetchMemos = async () => {
  const res = await axios.get<Memo[]>(`${API}/memos`);
  setMemos(res.data);
};

// 初回マウント時のみ取得。fetchMemos を deps に入れると無限ループになるため直接記述
useEffect(() => {
  axios.get<Memo[]>(`${API}/memos`).then(res => setMemos(res.data));
}, []);
```

- `[]`（空の配列）を第2引数に渡すと、「最初の1回だけ実行」という意味になります
- ここではアプリ起動時に「メモ一覧を取得する」処理を呼んでいます
- `fetchMemos` を依存配列に追加すると毎レンダーで再生成されてループになるため、初回だけは直接 `axios.get` を書いています

---

### TypeScript とは

**TypeScript** は JavaScript に「型」を追加した言語です。

```typescript
// 例：型があると間違いを防げる
const id: number = 42; // 数値しか入れられない
const title: string = "メモ"; // 文字列しか入れられない
```

- 変数や関数に「どんなデータが入るか」を明示します
- 間違った使い方をするとコンパイル時にエラーが出るため、バグを早期発見できます

---

### Vite とは（ビルドツール）

**Vite（ヴィート）** は開発サーバーを高速に起動するためのツールです。

```bash
npm run dev   # Vite の開発サーバーを起動 → http://localhost:5173
npm run build # 本番用にファイルをまとめる
```

- ファイルを変更すると、ブラウザが自動でリロードされます（HMR: Hot Module Replacement）
- `vite.config.ts` で設定を管理します

## 3. バックエンド：Node.js + Express

### Node.js とは

**Node.js** は JavaScript をブラウザ外（サーバー上）で動かすための実行環境です。

- 通常、JavaScript はブラウザでしか動きません
- Node.js を使うと、PC上・サーバー上でも JS を実行できます

### Express とは

**Express** は Node.js 向けの軽量な Web フレームワークです。「URL へのリクエストを受け取って、処理を返す」仕組みを簡単に書けます。

```javascript
// server/index.js（5〜12行目）
const express = require("express");
const app = express();
const PORT = process.env.PORT || 3001;

app.use(cors()); // クロスオリジン通信を許可
app.use(express.json()); // JSONを受け取れるようにする

app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
```

**ミドルウェア** とは: `app.use(...)` で登録する「すべてのリクエストに共通で適用する処理」のことです。

---

### CORS とは

**CORS（Cross-Origin Resource Sharing）** は、「異なるオリジン（ドメイン/ポート）からのアクセスを許可するか」のルールです。

```
フロントエンド: http://localhost:5173
バックエンド:   http://localhost:3001  ← ポートが違う！
```

ポートが違うと「別のオリジン」と見なされ、ブラウザがリクエストをブロックします。`cors()` ミドルウェアを使うとこれを解除できます。

---

### API ルートの書き方

```javascript
// server/index.js（23〜30行目）— GET（取得）
app.get("/api/memos", async (req, res) => {
  const { data, error } = await supabase
    .from("memos")
    .select("*")
    .order("created_at", { ascending: false });

  if (error) return res.status(500).json({ error: error.message });
  res.json(data); // 取得したデータをJSON形式で返す
});
```

| 部分                  | 意味                                                 |
| --------------------- | ---------------------------------------------------- |
| `app.get(...)`        | GET メソッドのリクエストを受け付ける                 |
| `'/api/memos'`        | URL のパス                                           |
| `async (req, res) =>` | リクエスト(req)を受け取り、レスポンス(res)を返す関数 |
| `res.json(data)`      | JSON 形式でデータを返す                              |
| `res.status(500)`     | HTTPステータスコード（500 = サーバーエラー）を返す   |

## 4. データベース：Supabase（PostgreSQL）

### Supabase とは

**Supabase** は PostgreSQL をベースにしたクラウドデータベースサービスです。無料で使い始められ、Web 画面からテーブルの作成・データ閲覧ができます。

### PostgreSQL とは

**PostgreSQL（ポスグレ）** は世界的に広く使われるオープンソースの関係データベース（RDB）です。データを「テーブル（表）」の形式で管理します。

### memos テーブルの構造

```sql
create table memos (
  id bigint generated always as identity primary key,
  title text not null,
  content text default '',
  image_url text,
  created_at timestamptz default now(),
  updated_at timestamptz
);
```

| カラム       | 型             | 意味                                     |
| ------------ | -------------- | ---------------------------------------- |
| `id`         | bigint（整数） | 各メモの一意な番号（自動採番）           |
| `title`      | text（文字列） | メモのタイトル（必須）                   |
| `content`    | text（文字列） | メモの本文（省略可、デフォルト空文字）   |
| `image_url`  | text（文字列） | 添付画像の公開URL（省略可）              |
| `created_at` | timestamptz    | 作成日時（自動で現在時刻が入る）         |
| `updated_at` | timestamptz    | 更新日時（更新時にサーバーがセットする） |

### Supabase JavaScript SDK の使い方

```javascript
// server/index.js
const { createClient } = require("@supabase/supabase-js");
const supabase = createClient(
  process.env.SUPABASE_URL, // プロジェクトのURL
  process.env.SUPABASE_KEY, // APIキー（anon public key）
);
```

SDK を使うと SQL を直接書かずにデータ操作できます：

```javascript
// 全件取得（SELECT * FROM memos ORDER BY created_at DESC）
await supabase
  .from("memos")
  .select("*")
  .order("created_at", { ascending: false });

// 1件追加（INSERT INTO memos (title, content) VALUES (...)）
await supabase.from("memos").insert([{ title, content }]).select();

// 1件更新（UPDATE memos SET title=..., content=... WHERE id=...）
await supabase.from("memos").update({ title, content }).eq("id", id).select();

// 1件削除（DELETE FROM memos WHERE id=...）
await supabase.from("memos").delete().eq("id", id);
```

## 5. フロント↔バック間の通信：REST API + Axios

### REST API とは

**REST API** は「URL + HTTP メソッド」でデータ操作を表現する設計スタイルです。

| HTTP メソッド | 操作     | このアプリでの URL      |
| ------------- | -------- | ----------------------- |
| GET           | 読み取り | `GET /api/memos`        |
| POST          | 作成     | `POST /api/memos`       |
| PUT           | 更新     | `PUT /api/memos/:id`    |
| DELETE        | 削除     | `DELETE /api/memos/:id` |

`:id` は「メモのID」が入る部分です（例: `/api/memos/5`）。

### Axios とは

**Axios** はブラウザから HTTP リクエストを送るためのライブラリです。

```tsx
// client/src/hooks/useMemos.ts — API通信はカスタムフックにまとめている
const API = "http://localhost:3001/api";

// GET（一覧取得）
const res = await axios.get(`${API}/memos`);

// POST（新規作成）
await axios.post(`${API}/memos`, { title, content, image_url });

// PUT（更新）
await axios.put(`${API}/memos/${id}`, { title, content, image_url });

// DELETE（削除）
await axios.delete(`${API}/memos/${id}`);
```

### async / await とは

**async / await** は非同期処理（時間がかかる処理）を読みやすく書くための構文です。

```javascript
// await なしだと…
axios.get("/api/memos").then((res) => {
  setMemos(res.data);
});

// await を使うと同期的に書ける
const res = await axios.get("/api/memos");
setMemos(res.data);
```

- `await` は「この処理が終わるまで次に進まない」という意味
- `async` は「この関数の中で await を使う」という宣言

## 6. 環境変数（.env）

**環境変数** とは、プログラムの外に切り出した設定値です。パスワードや API キーをコードに直接書くと危険なため、`.env` ファイルで管理します。

```javascript

```

## 7. CRUD とは

**CRUD** はデータ操作の基本4種類の頭文字です。

| 文字       | 操作     | SQL    | HTTP メソッド |
| ---------- | -------- | ------ | ------------- |
| **C**reate | 作成     | INSERT | POST          |
| **R**ead   | 読み取り | SELECT | GET           |
| **U**pdate | 更新     | UPDATE | PUT           |
| **D**elete | 削除     | DELETE | DELETE        |

このアプリはメモの CRUD をすべて実装した「基本的な Web アプリ」の典型例です。

## 8. データの流れを追ってみる（全体シナリオ）

### シナリオ：「新しいメモを追加する」

```
1. ユーザーがタイトル・内容を入力して「追加」ボタンをクリック

2. [フロントエンド] handleSubmit() が呼ばれる
   → axios.post('http://localhost:3001/api/memos', { title, content })

3. [HTTP通信] POST リクエストがサーバーへ送られる
   リクエストボディ: { "title": "買い物リスト", "content": "牛乳, 卵" }

4. [バックエンド] app.post('/api/memos', ...) が受け取る
   → supabase.from('memos').insert([{ title, content }])

5. [データベース] Supabase が memos テーブルに1行追加
   → INSERT INTO memos(title, content) VALUES('買い物リスト', '牛乳, 卵')

6. [バックエンド] 追加したデータを JSON で返す
   レスポンス: { "id": 3, "title": "買い物リスト", "content": "牛乳, 卵", ... }

7. [フロントエンド] fetchMemos() で最新の一覧を再取得

8. 画面にメモが表示される
```

## 9. 本プロジェクト固有の技術要素

ここでは、基本解説には含まれていないものの、このアプリで実際に使われている技術をまとめます。

---

### コンポーネント分割と props の受け渡し

`App.tsx` が肥大化するのを防ぐため、機能ごとにコンポーネントファイルを分割しています。

```
App.tsx（ルート）
├── Header.tsx        ← タイトル・テーマ切替・ユーザー名入力
├── MemoForm.tsx      ← メモ入力フォーム（新規追加・編集）
├── ViewToggle.tsx    ← グリッド/リスト切替ボタン
├── MemoCard.tsx      ← メモ1件のカード表示
└── Modal.tsx         ← 詳細表示のモーダル
```

各コンポーネントは必要なデータと関数を **props（プロップス）** として受け取ります。

```tsx
// client/src/App.tsx — MemoCard に必要な値とコールバックを渡す
<MemoCard
  memo={memo}           // 表示するメモデータ
  viewMode={viewMode}   // グリッド or リスト
  colors={colors}       // テーマ色
  onDetail={() => setDetailMemo(memo)}   // 詳細ボタンが押されたとき
  onEdit={() => form.startEdit(memo)}    // 編集ボタンが押されたとき
  onDelete={() => handleDelete(memo.id)} // 削除ボタンが押されたとき
/>
```

```tsx
// client/src/components/MemoCard.tsx — props を受け取って表示する
interface Props {
  memo: Memo;
  viewMode: 'grid' | 'list';
  colors: ThemeColors;
  onDetail: () => void;
  onEdit: () => void;
  onDelete: () => void;
}

export function MemoCard({ memo, viewMode, colors, onDetail, onEdit, onDelete }: Props) {
  // ...
}
```

`interface Props` でそのコンポーネントが受け取れる値の「形」を定義しています。親から渡された `onDetail` などの関数を呼ぶだけで、コンポーネント自身は state を持たないシンプルな作りです。

---

### カスタムフック（useForm）

**カスタムフック** とは、複数の state や処理をまとめて再利用しやすくしたオリジナルのフックです。名前を `use` で始める規則があります。

このアプリでは、フォームの入力・画像選択・送信処理を `useForm` にまとめています。

```tsx
// client/src/hooks/useForm.ts
export function useForm(addMemo, updateMemo, uploadImage, username) {
  const [title, setTitle] = useState('');
  const [content, setContent] = useState('');
  const [editingId, setEditingId] = useState<number | null>(null);
  const [imageFile, setImageFile] = useState<File | null>(null);
  const [imagePreview, setImagePreview] = useState<string | null>(null);
  const fileInputRef = useRef<HTMLInputElement>(null);

  // フォームを空に戻す（投稿・キャンセル後に呼ぶ）
  const reset = () => { ... };

  // 追加ボタン押下時の処理
  const handleSubmit = async () => { ... };

  // 既存メモをフォームに展開して編集モードに切り替える
  const startEdit = (memo: Memo) => { ... };

  return { title, setTitle, content, setContent, imagePreview, editingId,
           fileInputRef, handleImageChange, handleImageRemove, reset, handleSubmit, startEdit };
}
```

```tsx
// client/src/App.tsx — フックを呼ぶだけでフォームの全機能が使える
const form = useForm(addMemo, updateMemo, uploadImage, username);

// form.title, form.handleSubmit, form.startEdit ... のように使う
```

こうすることで `App.tsx` にフォーム関連のコードを書かなくて済み、それぞれのファイルの役割が明確になります。

---

### TypeScript の Interface（型定義）

このアプリでは `Memo` 型を `interface` で定義しています。これにより「メモオブジェクトはこのカラムで構成される」という形を明示しています。

```tsx
// client/src/types.ts — 型定義と日付ユーティリティを切り出して各ファイルから共用
export interface Memo {
  id: number;
  title: string;
  content: string;
  image_url?: string | null;  // ? は「あってもなくてもよい」という意味
  author?: string | null;     // 投稿者名（省略可）
  created_at: string;
  updated_at?: string;
}

// ISO 8601 文字列を "YYYY/MM/DD HH:mm" 形式に変換するユーティリティ
export function formatDate(dateStr: string) {
  const d = new Date(dateStr);
  return d.toLocaleString('ja-JP', { year: 'numeric', month: '2-digit', day: '2-digit', hour: '2-digit', minute: '2-digit' });
}
```

`?` をつけると「省略可能（optional）」な項目になります。`string | null` は「文字列か null のどちらか」という意味です。

`types.ts` には型定義だけでなく、日付フォーマットのような「型に関係するユーティリティ関数」もまとめています。コンポーネントファイルに書き散らすと管理が大変になるためです。

---

### useRef（DOM要素への参照）

**useRef** は「DOM の要素（HTMLタグ）を直接操作したい」ときに使うフックです。

```tsx
// client/src/hooks/useForm.ts — フォームの状態管理と一緒に useRef もここで管理
const fileInputRef = useRef<HTMLInputElement>(null);

// ファイル選択をリセットするとき
if (fileInputRef.current) fileInputRef.current.value = '';
```

ファイル選択ボタン（`<input type="file">`）は通常の state では値をリセットできないため、`useRef` で直接 `.value = ''` を代入してリセットしています。

`fileInputRef` はフォームの操作と密接に関係するため、`useForm` フックの中で管理し、コンポーネント（`MemoForm`）へ `ref` として渡しています。

### テーマ機能と localStorage

このアプリはカラーテーマを5種類（グリーン・ブルー・レッドなど）切り替えられます。テーマの選択状態・ユーザー名・表示モードはブラウザを閉じても維持されます。

#### localStorage とは

**localStorage** はブラウザ上に小さなデータを永続保存する仕組みです。サーバーに送らずブラウザ内だけに保存されます。

このアプリでは3つの設定を localStorage で管理しています：

```tsx
// client/src/App.tsx — 起動時にlocalStorageから読み込む（遅延初期化）
const [theme, setTheme] = useState<ThemeName>(
  () => (localStorage.getItem('memo-theme') as ThemeName) || 'green'
);
const [username, setUsername] = useState(
  () => localStorage.getItem('memo-username') || ''
);
const [viewMode, setViewMode] = useState<'grid' | 'list'>(
  () => (localStorage.getItem('memo-view') as 'grid' | 'list') || 'grid'
);

// 変更時に保存（テーマの例）
const handleThemeChange = (t: ThemeName) => {
  setTheme(t);
  localStorage.setItem('memo-theme', t); // ブラウザに保存
};
```

| キー           | 保存内容         |
| -------------- | ---------------- |
| `memo-theme`   | 選択中のテーマ名 |
| `memo-username`| 投稿者名         |
| `memo-view`    | 表示モード（grid / list） |

`useState` の初期値に関数を渡すと「初回レンダリング時だけ実行」されます。これを**遅延初期化**といいます。

#### テーマ定義の外部ファイル分離

色の定義は `client/src/styles.ts` に切り出されています。コンポーネントファイル（App.tsx）をスリムに保つための設計です。

```ts
// client/src/styles.ts
export const themes: Record<ThemeName, ThemeColors> = {
  green: {
    pageBg: '#edf7f1',
    btnPrimary: '#5aaa78',
    // ...
  },
  blue: { ... },
};
```

`Record<ThemeName, ThemeColors>` は「ThemeName のキーと ThemeColors の値の組み合わせ」という TypeScript の型です。

### モーダルダイアログ

メモの「詳細」ボタンを押すと、画面全体に重なる詳細表示（モーダル）が出現します。

```tsx
// client/src/components/Modal.tsx — 別ファイルのコンポーネントとして分離
export function Modal({ memo, onClose, colors }) {
  return (
    // オーバーレイ（背景の暗い部分）をクリックで閉じる
    <div onClick={onClose} style={{ position: 'fixed', inset: 0, ... }}>
      {/* stopPropagation でオーバーレイのonCloseへイベントが伝播するのを防ぐ */}
      <div onClick={e => e.stopPropagation()}>
        ...
      </div>
    </div>
  );
}
```

**ポイント**: `e.stopPropagation()` は「クリックイベントが親要素へ伝わるのを止める」処理です。これがないと、モーダル本体をクリックしたときも閉じてしまいます。

モーダルは `state` で表示/非表示を切り替えます：

```tsx
// client/src/App.tsx
// 詳細ボタン押下 → detailMemo に選択したメモをセット（MemoCard のコールバック経由）
<MemoCard onDetail={() => setDetailMemo(memo)} ... />

// detailMemo があればモーダルを表示（null のときは何も表示しない）
{detailMemo && <Modal memo={detailMemo} onClose={() => setDetailMemo(null)} colors={colors} />}
```

### 画像アップロード：multer と FormData

#### multer とは

**multer** はファイルアップロードを処理するための Express ミドルウェアです。通常の JSON では画像ファイルを送れないため、`multipart/form-data` という形式を使います。

```javascript
// server/index.js
const upload = multer({
  storage: multer.memoryStorage(),        // ファイルをメモリ上に保持
  limits: { fileSize: 5 * 1024 * 1024 }, // 5MB 制限
  fileFilter: (_req, file, cb) => {
    if (file.mimetype.startsWith('image/')) cb(null, true); // 画像のみ許可
    else cb(new Error('Images only'));
  },
});

// 画像アップロードエンドポイント
app.post('/api/upload', upload.single('image'), async (req, res) => {
  // req.file にアップロードされたファイルの情報が入る
});
```

#### FormData とは

**FormData** はブラウザ側でファイルを含むデータを組み立てるための Web 標準 API です。

```tsx
// client/src/hooks/useMemos.ts — 画像アップロードもカスタムフック内で処理
const formData = new FormData();
formData.append('image', file); // 'image' というキーでファイルを追加（サーバーのupload.single('image')と一致）
const res = await axios.post<{ url: string }>(`${API}/upload`, formData);
return res.data.url;
```

#### URL.createObjectURL（画像プレビュー）

ファイルをサーバーに送る前に、ブラウザ内でプレビュー表示するために使います。

```tsx
// client/src/App.tsx
setImagePreview(URL.createObjectURL(file)); // ローカルの一時URLを生成
```

実際のアップロードが起きているわけではなく、ブラウザ内のメモリ上にある画像を表示するための一時的な URL です。

### Supabase Storage（ファイル保存）

データベースには文字列しか保存できません。画像ファイルは **Supabase Storage** という別のストレージサービスに保存し、その公開 URL だけをデータベースに保存します。

```javascript
// server/index.js
// Supabase Storage にファイルをアップロード
const { error } = await supabase.storage
  .from('memo-images')     // バケット名（フォルダのようなもの）
  .upload(fileName, req.file.buffer, { contentType: req.file.mimetype });

// 公開URLを取得してデータベースに保存
const { data } = supabase.storage.from('memo-images').getPublicUrl(fileName);
res.json({ url: data.publicUrl });
```

#### service_role キーと RLS

Supabase には **RLS（Row Level Security）** という「誰がどのデータにアクセスできるか」を制限する仕組みがあります。ストレージの削除操作はこの制限の対象になるため、管理者権限を持つ `service_role` キーを使う別のクライアントを用意しています。

```javascript
// server/index.js
const supabaseAdmin = process.env.SUPABASE_SERVICE_KEY
  ? createClient(process.env.SUPABASE_URL, process.env.SUPABASE_SERVICE_KEY)
  : supabase;
```

> **注意**: `service_role` キーは強力な権限を持つため、絶対にフロントエンドには使わず、サーバー側のみで使います。

### CSS Grid によるカードレイアウト

メモ一覧は CSS Grid で並んでいます。画面幅に応じて自動的に列数が変わるレスポンシブなレイアウトです。

```tsx
// client/src/App.tsx
<div style={{
  display: 'grid',
  gridTemplateColumns: 'repeat(auto-fill, minmax(260px, 1fr))',
  gap: 16,
}}>
```

| 記述           | 意味                                              |
| -------------- | ------------------------------------------------- |
| `auto-fill`    | 画面幅に収まる分だけ列を自動生成する              |
| `minmax(260px, 1fr)` | 最小260px、余った幅は均等に割り当てる       |

これにより、幅の広い画面では3〜4列、スマホでは1列に自動変化します。

### 文字数制限と視覚的フィードバック

タイトル（50文字）・本文（500文字）には上限があり、残り文字数をリアルタイムで表示します。

```tsx
// client/src/App.tsx — 50文字を超えた入力を切り捨てる
onChange={e => setTitle(e.target.value.slice(0, 50))}

// 残り文字数の色を段階的に変える
color: title.length >= 45
  ? (title.length >= 50 ? '#e53e3e' : '#d97706')  // 45〜49字: 黄, 50字: 赤
  : colors.dateText                                 // 通常色
```

`slice(0, 50)` で50文字を超えた部分を切り捨て、`color` の三項演算子で残り文字数に応じて色を変えています。

## 10. 用語集

| 用語                  | 意味                                                       |
| --------------------- | ---------------------------------------------------------- |
| **フロントエンド**    | ブラウザ上で動くユーザー向けの画面部分                     |
| **バックエンド**      | サーバー上で動くデータ処理・API 提供部分                   |
| **コンポーネント**    | React における画面の部品単位                               |
| **state（ステート）** | React が管理する「変化するデータ」                         |
| **API**               | プログラム同士がやり取りするためのインターフェース         |
| **REST API**          | URL + HTTP メソッドでデータ操作を表現する設計スタイル      |
| **JSON**              | データを表すテキスト形式。`{"key": "value"}` の形式        |
| **HTTP メソッド**     | GET / POST / PUT / DELETE など、リクエストの種類           |
| **非同期処理**        | 結果を待たずに次の処理を進められる仕組み                   |
| **async / await**     | 非同期処理を同期的に書ける構文                             |
| **環境変数**          | コードの外に切り出した設定値（パスワードなど）             |
| **CORS**              | 異なるオリジン間の HTTP 通信を制御する仕組み               |
| **ミドルウェア**      | Express でリクエスト全体に適用する共通処理                 |
| **SDK**               | サービスを使いやすくするためのライブラリセット             |
| **CRUD**              | Create / Read / Update / Delete — データ操作の基本4種      |
| **RDB**               | 関係データベース。テーブル形式でデータを管理する           |
| **プライマリキー**    | テーブルの各行を一意に識別するカラム（ここでは `id`）      |
| **HMR**               | Hot Module Replacement。ファイル変更時にブラウザを自動更新 |
